# репо + нейронка vs context7 — что лучше для вайбкодинга плагинов?

> короткий ответ: для плагинов ExteraGram/AyuGram — репо + нейронка выигрывает. но не потому что context7 плохой, а потому что они решают разные задачи. читай дальше.

---

## что такое context7

context7 — это MCP-сервер от upstash. его задача простая: когда ты пишешь промпт и добавляешь `use context7`, он автоматически вытаскивает актуальную документацию нужных библиотек прямо в контекст нейронки.

проблема, которую он решает: нейронки обучены на данных с определённой датой среза. если библиотека обновилась после этой даты — нейронка будет галлюцинировать старые API, несуществующие методы и устаревший синтаксис. context7 это фиксит, подтягивая живую доку из репозиториев.

звучит круто. и для своей задачи — это реально крутой инструмент.

---

## почему context7 не поможет с плагинами для ExteraGram/AyuGram

вся проблема в одном: **context7 знает только публичные библиотеки с документацией**.

когда ты пишешь плагин для ExteraGram, ты работаешь с такими вещами:

- `base_plugin.BasePlugin` — базовый класс плагина
- `base_plugin.MethodHook` — хуки на Java-методы
- `hook_utils.find_class`, `get_private_field`, `set_private_field` — рефлекшн внутренностей Telegram
- `android_utils.run_on_ui_thread` — безопасный выход в UI-поток
- `ui.bulletin.BulletinHelper` — показ уведомлений
- `client_utils.get_last_fragment`, `get_messages_controller` — доступ к состоянию клиента
- `org.telegram.ui.Stars.SuperRipple` — внутренний класс самого Telegram
- `org.telegram.messenger.AndroidUtilities` — утилиты Telegram

**ни одного из этих классов нет в context7.** это не публичные библиотеки с документацией на github — это внутренности конкретного форка Telegram, которые нигде официально не задокументированы.

если попросить нейронку написать плагин с `use context7` — она не найдёт ничего полезного и будет угадывать API наугад. результат: нерабочий код.

---

## что даёт это репо

репо закрывает именно тот пробел, который context7 закрыть не может: **реальные рабочие паттерны для конкретного plugin API ExteraGram**.

### 1. рабочие примеры кода

в репо лежат готовые `.plugin` файлы с рабочим кодом. нейронка видит реальные примеры того, как устроен плагин: как регистрировать хуки, как работать с Java-объектами через Jython, как правильно чистить ресурсы при выгрузке.

### 2. детальные анализы

каждый анализ объясняет не просто *что* написано в коде, а *почему именно так*. нейронка, получив анализ в контексте, понимает:
- почему `before_hooked_method`, а не `after_hooked_method`
- почему `getRawX/getRawY`, а не `getX/getY`
- почему токен дедупликации через `getDownTime()`
- почему `run_on_ui_thread` обязателен при работе с `View`

без этого понимания нейронка будет делать технически похожий, но неработающий или нестабильный код.

### 3. мини-туториал для AI

файл `mini-tutorial-for-AI.md` даёт нейронке базовый контекст по plugin API ещё до того, как ты начал писать промпт. это как системный промпт, только в файле.

### 4. живая дока из практики

официальная дока по плагинам ExteraGram — минималистичная. большая часть знаний о том, как реально работает API, существует только в виде рабочего кода. этот репо и есть та самая живая дока, написанная из практики, а не из теории.

## где context7 реально полезен

если в твоём плагине есть код, который использует стандартные библиотеки — context7 там работает отлично.

примеры где он поможет:
- работа с `threading`, `asyncio`, стандартной библиотекой Python
- HTTP-запросы через `requests` или `httpx`
- парсинг JSON, работа с файлами
- стандартный Android API (`android.view`, `android.os`, `android.graphics`) — там context7 может подтянуть актуальную доку

то есть если ты пишешь, например, часть плагина, которая делает сетевой запрос к внешнему API — context7 там уместен. но как только дело доходит до `BasePlugin`, хуков и внутренностей Telegram — он бесполезен.

## итоговое сравнение

| | репо + нейронка | context7 + нейронка |
|---|---|---|
| plugin API ExteraGram/AyuGram | ✅ полное покрытие | ❌ нет доки |
| внутренние классы Telegram | ✅ есть примеры и анализы | ❌ не знает |
| хуки, рефлекшн, Java-объекты | ✅ разобраны детально | ❌ нет |
| стандартные Python-библиотеки | ⚠️ нейронка и так знает | ✅ актуальная дока |
| стандартный Android API | ⚠️ нейронка и так знает | ✅ актуальная дока |
| популярные фреймворки (React и т.д.) | ❌ не про это | ✅ отлично |
| порог входа | низкий — просто скинь файлы в контекст | средний — нужна настройка MCP |

## вывод

context7 — хороший инструмент, но для другой задачи. он создан для разработки на популярных публичных библиотеках, где есть живая документация.

плагины ExteraGram/AyuGram — нишевая история с закрытым внутренним API. для этой задачи единственный рабочий подход: скидывать нейронке рабочие примеры кода и детальные анализы. именно для этого и существует этот репо.

**формула простая: нейронка + файлы из этого репо в контексте = рабочий плагин. нейронка + context7 для плагинов = галлюцинации.**
